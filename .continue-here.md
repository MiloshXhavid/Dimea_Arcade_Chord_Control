## Suspected bugs to fix

### Fix A — Looper stop resetNoteCount() (lines 824, 856)
Remove `resetNoteCount()` from both looper stop detection and looper reset paths.
The targeted per-voice decrements in the loop above already handle the looper notes correctly.
resetNoteCount() additionally zeroes live pad counts → hang when pads released after looper stops.

### Fix B — Sub toggle-on uses stale heldPitch_
Line ~998: `const int subPitch = juce::jlimit(0, 127, heldPitch_[v] - 12);`
This runs BEFORE heldPitch_[v] is updated at line 1032.
Fix: move sub toggle detection AFTER line 1032, or use freshPitches[v] instead of heldPitch_[v].

### Fix C — If user has sub-octave ON
The stale subPitch in toggle-on can get stuck if a re-trigger fires in the same block.
The fix is Fix B above.
