  <action>
Make exactly two edits to CMakeLists.txt:

**Edit 1 — Add global static CRT setting.**
Insert the following two lines immediately after the `project(ChordJoystick VERSION 1.0.0)` line and before the `include(FetchContent)` line:

```cmake
# Static CRT — must precede all FetchContent calls so JUCE internals also get /MT
set(CMAKE_MSVC_RUNTIME_LIBRARY "MultiThreaded$<$<CONFIG:Debug>:Debug>")
```

This must appear BEFORE `include(FetchContent)` — if it comes after `FetchContent_MakeAvailable(juce)`, JUCE's internal static library targets will be compiled with the default /MD CRT, causing LNK2038 mismatch errors at link time.

**Edit 2 — Pin JUCE to 8.0.4.**
Replace the existing FetchContent_Declare block for juce:

Old block (lines 7-11):
```cmake
FetchContent_Declare(
    juce
    GIT_REPOSITORY https://github.com/juce-framework/JUCE.git
    GIT_TAG        origin/master
)
```

New block:
```cmake
FetchContent_Declare(
    juce
    GIT_REPOSITORY https://github.com/juce-framework/JUCE.git
    GIT_TAG        8.0.4        # commit 51d11a2be6d5c97ccf12b4e5e827006e19f0555a — Nov 18 2024
    GIT_SHALLOW    TRUE
)
```

`GIT_SHALLOW TRUE` is safe here because `8.0.4` is a named annotated tag. Do NOT use the bare SHA with GIT_SHALLOW TRUE (that causes a CMake error).

Do NOT modify the SDL2 FetchContent block, the juce_add_plugin call, target_sources, target_compile_definitions, or target_link_libraries — all of those are already correct.

<task type="auto">
  <name>Task 2: Fix PluginProcessor.cpp — Ableton dummy bus + updated bus layout validator, then build</name>
  <files>Source/PluginProcessor.cpp</files>
  <action>
Make exactly two edits to Source/PluginProcessor.cpp:

**Edit 1 — Update the constructor BusesProperties() call.**
Current constructor (lines 138-142):
```cpp
PluginProcessor::PluginProcessor()
    : AudioProcessor(BusesProperties()),
      apvts(*this, nullptr, "ChordJoystick", createParameterLayout())
{
}
```

Replace with:
```cpp
PluginProcessor::PluginProcessor()
    : AudioProcessor(
          juce::PluginHostType().isAbletonLive()
              ? BusesProperties().withOutput("Output", juce::AudioChannelSet::stereo(), false)
              : BusesProperties()),
      apvts(*this, nullptr, "ChordJoystick", createParameterLayout())
{
}
```

The `false` third argument to `withOutput` marks the bus as disabled-by-default — it carries no audio; it exists only to satisfy Ableton's requirement that at least one audio bus be present.

`juce::PluginHostType().isAbletonLive()` is a runtime check so the plugin still presents zero buses in Reaper, Bitwig, and standalone mode.

**Edit 2 — Update isBusesLayoutSupported to accept 0 or 2 output channels.**
Current implementation (lines 155-161):
```cpp
bool PluginProcessor::isBusesLayoutSupported(const BusesLayout& layouts) const
{
    // MIDI-only plugin: no audio buses
    if (layouts.getMainInputChannelCount()  != 0) return false;
    if (layouts.getMainOutputChannelCount() != 0) return false;
    return true;
}
```

Replace with:
```cpp
bool PluginProcessor::isBusesLayoutSupported(const BusesLayout& layouts) const
{
    // No audio input ever
    if (layouts.getMainInputChannelCount() != 0) return false;
    // Allow zero output (Reaper/Bitwig/standalone) or stereo dummy output (Ableton Live)
    const int outCh = layouts.getMainOutputChannelCount();
    if (outCh != 0 && outCh != 2) return false;
    return true;
}
```
