## Decisions Made

- Item ID 20 for lfoX/Y (19 named CCs already at 1-19); ID 26 for filterX/Y (25 named modes at 1-25) — consistent with backend `kCustomCcDestIdx=19` / `kCustomFilterModeIdx=25`
- `filterXCustomCcParamId_` / `filterYCustomCcParamId_` strings (not raw pointers) swapped on INV toggle — simpler and safe because INV fires at most once per timer tick
- timerCallback visibility guard prevents calling `resized()` on every 30 Hz tick — only fires when combo is custom-mode but label isn't yet visible (preset load scenario)
