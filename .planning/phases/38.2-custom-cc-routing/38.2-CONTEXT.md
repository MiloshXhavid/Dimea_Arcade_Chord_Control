# Phase 38.2: Custom CC Routing

## Goal
Allow users to assign any arbitrary MIDI CC number (0–127) to the LFO CC Dest and Left Stick X/Y Mode dropdowns, instead of being limited to the predefined named list.

## Background
The CC assignment dropdowns (LFO X CC Dest, LFO Y CC Dest, Left Stick X Mode, Left Stick Y Mode) expose a fixed list of named CCs. Users need to route to any CC number their synth/device uses. A first attempt using IncDecButtons inline was too cramped and unreadable — this phase replaces that with a proper editable text field approach.

## Decisions

### A — UI Mechanism
- "Custom CC..." is added as the **last item** in each combo's list
- When selected, the combo **shrinks** to make room for an **editable text field** inline in the same row
- The text field shows the **last named CC number** that was active before switching to custom
- User types a number and presses Enter to confirm
- While custom is active the combo label shows the number dynamically: `"CC 42"`

### B — Which dropdowns
All four dropdowns get "Custom CC...":
- `lfoXCcDestBox_` — LFO X CC Dest
- `lfoYCcDestBox_` — LFO Y CC Dest
- `filterXModeBox_` — Left Stick X Mode
- `filterYModeBox_` — Left Stick Y Mode

"Custom CC..." is appended **last** in each list — existing item indices must not shift (there are connections to them in `PluginProcessor.cpp`).

### C — Display label
When custom is active the combo shows `"CC [n]"` dynamically (e.g. `"CC 42"`), not a static "Custom CC" label.

## Constraints
- All four combos are ~150–166px wide, 22px tall — inline text field is viable
- Existing item IDs/indices must not change — "Custom CC..." appends after current last item
- CC value clamped silently to 0–127 (no warning dialog)

## Key Files
- `Source/PluginEditor.cpp` — combo creation/population (lines ~2898–2956 filterX/Y, ~3102–3117 lfoXCcDest, ~3258–3273 lfoYCcDest) and setBounds (resized block)
- `Source/PluginProcessor.cpp` — CC lookup dispatch using `kFilterCcNums[]` and `lfoXCcDest`/`lfoYCcDest` APVTS params
- `Source/PluginEditor.h` — member declarations for new text field components

## Storage
4 separate APVTS float params (one per combo) storing the custom CC number (0–127).
Names TBD by planner — suggest `lfoXCustomCc`, `lfoYCustomCc`, `filterXCustomCc`, `filterYCustomCc`.
