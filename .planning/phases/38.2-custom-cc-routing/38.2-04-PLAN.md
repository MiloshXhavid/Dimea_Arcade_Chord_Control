<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Custom CC routing for all four dropdowns (LFO X CC Dest, LFO Y CC Dest, Left Stick X Mode, Left Stick Y Mode). Each dropdown has a new "Custom CC..." last item. When selected, an inline editable text field appears beside the combo. Typing a number + Enter routes that MIDI CC number and updates the combo label to "CC [n]".
  </what-built>
  <how-to-verify>
Load the plugin in your DAW. Open a MIDI monitor on the same track.

**Test 1 — LFO X CC Dest custom entry:**
1. Open LFO X panel. Set LFO X CC Dest to "Custom CC..." — a small text field should appear to the right of the combo.
2. The text field shows the previous CC number (default 74). Type "42" and press Enter.
3. The combo label should now show "CC 42" (not "Custom CC...").
4. Enable LFO X with Level > 0 and Shape = Sine. Confirm the MIDI monitor shows CC 42 being sent (not CC 74).
5. Select a named CC (e.g. "CC74 - Filter Cut") from the dropdown — the text field should disappear.

**Test 2 — filterXMode (Left Stick X Mode) custom entry:**
1. In the Filter Mod section, set Left Stick X Mode to "Custom CC...".
2. A text field should appear. Type "100" and press Enter. Combo shows "CC 100".
3. Move the gamepad left stick X (or use mouse filter mode). Confirm MIDI monitor shows CC 100 being sent.
4. Select "CC71 - Resonance" — text field disappears.

**Test 3 — Preset save/load:**
1. Set LFO X CC Dest to "Custom CC...", type "42", press Enter.
2. Save preset in your DAW. Reload the preset.
3. Confirm combo shows "CC 42" and the text field is visible with "42" in it.

**Test 4 — INV button swap:**
1. Set Left Stick X Mode to "Custom CC..." and type "55". Set Left Stick Y Mode to "Custom CC..." and type "66".
2. Toggle the INV button. Verify that after toggling, what was the X column now routes to CC 66 (the Y value moved there) and Y routes to CC 55.
3. Toggle INV back — routing should return to original.

**Test 5 — All four combos (quick check):**
Confirm "Custom CC..." is present in lfoYCcDestBox_ and filterYModeBox_ as well and that the same inline-text behavior works.
  </how-to-verify>
