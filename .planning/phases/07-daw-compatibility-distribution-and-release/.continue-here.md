<completed_work>

- Diagnosed root cause 1: VST3_CATEGORIES "Fx|MIDI Effect" was a single pipe-delimited string
  in CMake, causing JUCE to prepend an extra "Fx" → "Fx|Fx|MIDI Effect" baked into old DLL.
  Fixed: changed to VST3_CATEGORIES "Fx" "MIDI Effect" (two CMake list items) → clean "Fx|MIDI Effect".

- Diagnosed root cause 2: Ableton 11.3.43 classifies the plugin as `audiofx` (not `midifx`)
  due to the "Fx" prefix. When loading as an audio effect, it requires an audio input bus.
  Plugin had BusesProperties() (empty) → no audio bus → load fails with "no valid audio input bus".

- Fixed: Added optional stereo audio buses to BusesProperties (inactive by default):
    .withInput ("Input",  juce::AudioChannelSet::stereo(), false)
    .withOutput("Output", juce::AudioChannelSet::stereo(), false)
  Updated isBusesLayoutSupported to accept (0+0) OR (2+2) — Ableton deactivates buses
  before loading on a MIDI track; with buses declared, activateBus() returns kResultTrue.

<decisions_made>

- VST3_CATEGORIES "Fx" "MIDI Effect" (two items) is correct JUCE CMake syntax to produce
  "Fx|MIDI Effect" without duplication. Single string "Fx|MIDI Effect" produces "Fx|Fx|MIDI Effect".

- Ableton 11 reads sub-categories from the compiled DLL (via VST3 factory call), NOT from
  moduleinfo.json. Fingerprint = size:timestamp. Changing only moduleinfo.json is insufficient.

- Ableton 11 classifies "Fx|MIDI Effect" as `audiofx` (not `midifx`). This is Ableton's
  behavior — it requires audio buses for anything in the "Fx" category. The `midifx`
  classification may not be achievable via sub-categories alone in Ableton 11.3.x.

- Adding optional audio buses (inactive by default) allows Ableton to satisfy its audio-bus
  requirement, then deactivate them for MIDI-track use. This is the correct workaround.
  processBlock already calls audio.clear() so silent audio passthrough is safe.

- isMidiEffect()=true kept as-is. JUCE VST3 wrapper reads bus count from BusesProperties,
  not from isMidiEffect(). Declaring optional buses is compatible with isMidiEffect()=true.
</decisions_made>
