**Step 1 — Create installer/ directory and write the .iss script.**

Create the file `installer/DimaChordJoystick-Setup.iss` at the project root with this
exact content (adjust relative paths from within the `installer/` directory — the Source
path must be relative to the .iss file location, so use `..` to reference the build output):

```iss
; DimaChordJoystick-Setup.iss
; Inno Setup 6.7.1 installer for ChordJoystick VST3 v1.0.0
; Place this file in the installer/ subdirectory of the project root.

[Setup]
AppName=ChordJoystick
AppVersion=1.0.0
AppPublisher=Dima Xhavid
AppPublisherURL=https://gumroad.com
OutputBaseFilename=DimaChordJoystick-Setup
OutputDir=Output
DefaultDirName={commoncf64}\VST3\ChordJoystick.vst3
LicenseFile=..\LICENSE.txt
PrivilegesRequired=admin
PrivilegesRequiredOverridesAllowed=dialog
DisableDirPage=yes
Compression=lzma
SolidCompression=yes
UninstallDisplayName=ChordJoystick VST3

[Files]
; Copies the entire .vst3 bundle (directory tree) to the standard VST3 system path.
; Source ends in \* so Inno Setup recurses into the directory.
; DestDir creates ChordJoystick.vst3\ at the destination.
Source: "..\build\ChordJoystick_artefacts\Release\VST3\ChordJoystick.vst3\*"; \
    DestDir: "{commoncf64}\VST3\ChordJoystick.vst3"; \
    Flags: ignoreversion recursesubdirs createallsubdirs

[UninstallDelete]
; Ensure the whole .vst3 bundle directory is removed on uninstall.
Type: filesandordirs; Name: "{commoncf64}\VST3\ChordJoystick.vst3"

; ── Code-signing placeholder ──────────────────────────────────────────────────
; Uncomment and fill in when a certificate is acquired:
; [Code]
; procedure CurStepChanged(CurStep: TSetupStep);
; var ResultCode: Integer;
; begin
;   if CurStep = ssPostInstall then begin
;     Exec('signtool.exe',
;       'sign /tr http://timestamp.digicert.com /td sha256 /fd sha256 /a ' +
;       '"' + ExpandConstant('{commoncf64}\VST3\ChordJoystick.vst3\Contents\x86_64-win\ChordJoystick.vst3"'),
;       '', SW_HIDE, ewWaitUntilTerminated, ResultCode);
;   end;
; end;
```

**Key design decisions in this script (per CONTEXT.md and RESEARCH.md):**
- `OutputBaseFilename=DimaChordJoystick-Setup` — produces exactly `DimaChordJoystick-Setup.exe` (LOCKED)
- `AppVersion=1.0.0` (LOCKED — matches CMakeLists.txt `project(ChordJoystick VERSION 1.0.0)`)
- `PrivilegesRequired=admin` + `PrivilegesRequiredOverridesAllowed=dialog` — gives user
  the system-wide vs per-user choice at the first screen (system-wide is default, per CONTEXT.md)
- `LicenseFile=..\LICENSE.txt` — produces the license agreement wizard page (LOCKED decision:
  wizard includes a license agreement page); LICENSE.txt is copied from the extension-less LICENSE at project root
- `DisableDirPage=yes` — VST3 path is standardised; no user browsing needed
- `Source: "..\build\...\*"` with `Flags: recursesubdirs createallsubdirs` — copies entire
  bundle tree; without these flags only the DLL would be copied (no moduleinfo.json)
- `[UninstallDelete]` with `filesandordirs` — ensures clean removal including directory node
- No start menu shortcuts, no desktop icons, no README popup, no browser launch (LOCKED)
- Code-signing placeholder commented out (LOCKED — no cert for v1)

  <how-to-verify>
Test the installer on a **clean Windows machine** (one without Visual Studio, JUCE, or the
MSVC build tools installed). This validates the static CRT setting and confirms musicians
can install the plugin without dev dependencies.

**Steps:**

1. Copy `installer\Output\DimaChordJoystick-Setup.exe` to the clean machine
   (USB drive, file share, or cloud upload).

2. Run the installer:
   - SmartScreen will show "Windows protected your PC" (unsigned, expected)
   - Click "More info" → "Run anyway"
   - Complete the wizard (welcome → install → done)

3. Confirm VST3 installed:
   - `C:\Program Files\Common Files\VST3\ChordJoystick.vst3\` directory exists
   - `Contents\x86_64-win\ChordJoystick.vst3` (the DLL) exists inside it

4. Load in a DAW on the clean machine:
   - In Reaper: rescan plugins → ChordJoystick appears → load on a track → plugin
     opens without crash → basic UI visible (joystick pad, touchplates, knobs)
   - OR in Ableton 11/12: rescan MIDI effects → ChordJoystick appears in MIDI Effects
     browser → drag to MIDI track → plugin opens without crash

5. Quick functional check:
   - Move the joystick pad → no crash
   - Press a touchplate → MIDI note-on observed (MIDI monitor or synth responds)
   - Release touchplate → note-off observed

6. Confirm Add/Remove Programs:
   - Windows Settings → Apps → search "ChordJoystick" → entry present

**Expected outcome:** Plugin installs, loads in DAW, and produces MIDI without errors on a
machine with no dev tools installed.

**If it fails:**
- If plugin crashes on load: likely runtime DLL issue — check that static CRT is actually
  compiled in (CMakeLists.txt: `CMAKE_MSVC_RUNTIME_LIBRARY "MultiThreaded"`). Rebuild with
  explicit `/MT` flag confirmation.
- If MIDI is silent: check voice MIDI channel routing in plugin UI (default channels 1-4)
  and confirm DAW/synth is listening on those channels.
- If "No types found" in DAW: bundle structure wrong — confirm `moduleinfo.json` and the
  DLL are both present in the installed directory tree.
  </how-to-verify>
