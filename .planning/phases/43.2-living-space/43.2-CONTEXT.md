# Phase 43.2 — Living Space: Context

## Goal
Make the joystick pad feel alive in idle (non-warp) mode from a "pilot floating through space" perspective. Every effect is extremely subtle — the space should feel like it breathes, not like a screensaver. Active only when warp is OFF (warpT_ == 0).

---

## Decisions

### A — Stars Feel Alive

**1. Star twinkle**
- Each star flickers in brightness independently using a slow sinusoidal oscillation + small LCG random phase offset per star
- Amplitude: ±8–12% of base alpha — imperceptible on a single frame, felt over time
- Frequency: very slow, ~0.3–0.8 Hz per star (random per star, never synchronized)
- No color shift — brightness only

**2. Shooting stars / micrometeorites**
- One streak every ~20s on average (Poisson-ish: random interval 12–30s)
- Short fast streak, 40–80px long, 1–1.5px wide, white with alpha fade-in/out over ~0.3s total
- Random angle, random position, crosses the pad (does not need to traverse full width)
- Very faint — peak alpha ~0.5, not a bright flash

**3. Parallax depth layers + unified flow direction**

**Core fix — direction:** Current stars have a random direction per star (snow globe feel). Replace with a unified flow model:
- All stars share a dominant drift direction — slow upward = flying forward through space
- Per-star angle variation: ±15° from dominant direction (organic, not robotic)
- Speed is unchanged — only the direction model changes

**Joystick coupling — accumulated heading (steering, not pointing):**
- The joystick deflection **rotates the drift heading over time** — it does NOT directly map joystick position to direction
- When the joystick snaps back to center, the heading stays at whatever angle it reached — no rubber-band snap
- Joystick right → heading rotates clockwise gradually; release → heading holds
- Full 360° rotation possible — pilot can turn around and drift "backward"
- Turn rate: ~15°/second at full deflection (gentle, not twitchy)
- Read deflection from `displayCx_/displayCy_` relative to pad center — use as a signed turn rate input
- New member: `float driftHeading_` (accumulated angle in radians, starts at -π/2 = upward/forward)

**Parallax depth layers** (speed stratification by r):
  - Near (r > 1.4): drift at full speed
  - Mid (r 0.8–1.4): drift at ~60% speed
  - Far (r < 0.8): drift at ~30% speed

**Implementation:** At star generation, store `baseAngle` (dominant direction ± 15° variation). Each frame, compute `effectiveAngle = driftHeading_ + s.baseAngle`. Apply parallax speed multiplier. Replace current torus-wrap position update with respawn logic (see below).

**Star boundary handling — respawn with fade-in (replaces torus wrap):**
- When a star exits any edge (`x < 0 || x > 1 || y < 0 || y > 1`): respawn at a random (x, y) position anywhere within the pad, reset `twinklePhase` to a new random value
- On respawn, start with `alpha = 0` and fade in over ~0.5s (~30 ticks at 60Hz)
- Per-star fade-in counter: `uint8_t fadeIn` (0 = fully faded in, counting down from 30 on respawn)
- This is direction-agnostic — works regardless of drift heading, depth layer, or future additions
- Replaces the current 4-line torus wrap entirely

### C — Universe Has Depth

**4. Nebula gas clouds**
- 2–3 soft elliptical blobs, purple/teal palette (e.g. `0x18B060FF` teal, `0x3020A0FF` violet, `0x602090FF` deep purple — all at very low alpha ~0.04–0.07)
- Each blob: large (80–180px radius), rendered as radial gradient fill
- Move extremely slowly — full pad crossing in ~90–120s
- Wrap around edges (torus-style)
- Rendered below the starfield layer

**5. Background hue drift**
- The base background fill color (`Clr::bg` = `0xFF131525`) shifts its hue very slowly over time
- Range: ±8 hue degrees from base, full cycle ~120s
- Implementation: in `paint()`, compute `hueOffset = sin(Time::getMillisecondCounterHiRes() / 120000.0 * twoPi) * 8.0` and apply to bg fill
- Extremely subtle — noticeable only if you stare at the background, not during active play

## Warp Mode Compatibility (Phase 42)

Each effect must respect `warpT` (0 = idle, 1 = full warp):

| Effect | During warp |
|--------|-------------|
| Twinkle | Multiply twinkle offset by `(1.0f - warpT)` — fades out as warp ramps in |
| Shooting stars | No new spawns when `warpT > 0`; fade any in-flight streak with `(1.0f - warpT)` |
| Parallax | **Only governs idle lateral drift speed.** Phase 42 freeze rules take over during warp: `r > 1.2` stars freeze regardless of parallax layer. No conflict — parallax multiplier is simply overridden when the freeze condition is met. |
| Nebula clouds | Fade alpha with `(1.0f - warpT)` — clouds dissolve as warp engages, naturally |
| Hue drift | Continues during warp — Phase 42 doesn't touch bg fill color, no interaction |

**Parallax boundary note:** Phase 42 freezes `r > 1.2` during warp. Phase 43.2 puts the Near/fast layer at `r > 1.4`. Stars in the `r 1.2–1.4` band move fast in idle but freeze in warp — this is intentional and consistent with Phase 42's "large stars = stationary nebula" decision. Do not change the Phase 42 freeze threshold.

## Out of Scope
- No cockpit/HUD overlay (B)
- No cursor breathing or gravitational lensing (D)
- No new APVTS parameters — purely visual, no user controls

## Code Context

### Existing starfield infrastructure (JoystickPad, Phase 31)
- Stars stored in `std::vector<Star>` with `{ x, y, r, speed, alpha }` or similar
- `spaceBgBaked_` is a pre-baked background image — nebula clouds and hue drift must be drawn **live** in `paint()`, not baked
- Warp state: `warpT_` float (0.0 = idle, 1.0 = full warp) — use this to crossfade living-space effects out as warp ramps in
- Timer: JoystickPad repaints at ~60 Hz via the editor's `Timer`

### New members needed (JoystickPad)
- Per-star twinkle phase: add `float twinklePhase` to `StarDot` struct
- Per-star base angle: add `float baseAngle` to `StarDot` struct (replaces baked `vx/vy` at generation time)
- Per-star fade-in: add `uint8_t fadeIn` to `StarDot` struct (0 = fully visible, counts down from 30 on respawn)
- New pad member: `float driftHeading_` (accumulated steering angle, init = -π/2 = upward)
- Shooting star state: `struct ShootingStar { float x,y, angle, progress, interval; }` — single instance, interval randomized on each completion
- Nebula blobs: `struct NebulaBlob { float x, y, rx, ry, vx, vy, hue, alpha; }` × 3
- No new APVTS params — all seeded from existing LCG or `juce::Random`

### Parallax + flow direction
- Each tick, update heading: `driftHeading_ += turnRate * joystickDeflectionX` where `joystickDeflectionX = (displayCx_ - padCenterX) / (padWidth * 0.5f)` (normalized -1..+1)
- `turnRate = 0.004f` radians/tick at full deflection (~15°/s at 60Hz)
- When joystick centered (deflection ≈ 0): `driftHeading_` unchanged — heading persists
- Each star moves: `effectiveAngle = driftHeading_ + s.baseAngle` (baseAngle is the ±15° individual variation)
- Apply depth multiplier: `depthMult = (s.r > 1.4f) ? 1.0f : (s.r > 0.8f) ? 0.6f : 0.3f`
- `s.x += cos(effectiveAngle) * s.speed * depthMult`
- `s.y += sin(effectiveAngle) * s.speed * depthMult`
