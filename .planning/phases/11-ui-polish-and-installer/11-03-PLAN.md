```cpp
// Looper playback position for UI progress bar (read by message thread — lock-free float atomic)
double getLooperPlaybackBeat()  const { return looper_.getPlaybackBeat();      }
double getLooperLengthBeats()   const { return looper_.getLoopLengthBeats();   }
```

These are inline accessors delegating to the private `looper_` member. `getPlaybackBeat()` returns an `std::atomic<float>` read (lock-free on all targets). `getLoopLengthBeats()` is not atomic — it reads `subdiv_` and `loopBars_` which are both `std::atomic<int>`, so reading from the message thread is safe.

The `looper_` member is private in PluginProcessor — these accessors are the only correct approach (do NOT make `looper_` public; do NOT access it directly from PluginEditor).

**Part B — paint(): Draw the position bar**

In `paint()`, after the `drawSectionPanel(...)` calls added in Plan 01, add the position bar drawing:

```cpp
// ── Looper position bar ───────────────────────────────────────────────────
if (!looperPositionBarBounds_.isEmpty())
{
    const auto barB = looperPositionBarBounds_.toFloat();

    // Background track
    g.setColour(Clr::gateOff);
    g.fillRoundedRectangle(barB, 2.0f);

    // Filled sweep — only when looper is playing or recording
    if (proc_.looperIsPlaying() || proc_.looperIsRecording())
    {
        const double len  = proc_.getLooperLengthBeats();
        const double beat = proc_.getLooperPlaybackBeat();
        if (len > 0.0)
        {
            const float fraction = juce::jlimit(0.0f, 1.0f, static_cast<float>(beat / len));
            if (fraction > 0.0f)
            {
                g.setColour(Clr::gateOn);
                g.fillRoundedRectangle(barB.withWidth(barB.getWidth() * fraction), 2.0f);
            }
        }
    }
}
```

Do NOT use `juce::ProgressBar` — it has an internal timer. This is a plain `paint()` draw.
Do NOT add a new timer — the existing 30 Hz `timerCallback()` drives the repaint.

**Part C — timerCallback(): Add partial repaint for the position bar**

```cpp
// Partial repaint for looper position bar (driven by this 30 Hz timer)
```

This ensures `PluginEditor::paint()` is called at 30 Hz in the bar's region, keeping the progress sweep smooth. Without this line, the bar would only repaint when child components invalidate the editor (unreliable and infrequent).
