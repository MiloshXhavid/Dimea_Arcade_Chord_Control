<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Three-branch Random Free sync matrix + burst semantics in TriggerSystem. Behavior depends on RND SYNC toggle and DAW Sync state:
    - RND SYNC OFF → truly random Poisson intervals (no grid alignment)
    - RND SYNC ON + DAW stopped → internal free-tempo clock grid at selected subdivision
    - RND SYNC ON + DAW playing → DAW beat grid at selected subdivision
    - Probability knob now controls burst size (0% = silence, 100% = 64-note rapid burst)
  </what-built>
  <how-to-verify>
    Rescan VST3 in your DAW and load the plugin on a MIDI track routed to a MIDI monitor.

    Set one voice to trigger source "Random Free". Use Population = 4, Probability = 50% as a starting point.

    **Test 1 — RND-08: RND SYNC OFF produces irregular intervals**
    1. Ensure RND SYNC is OFF (button not highlighted)
    2. Ensure DAW is STOPPED or DAW Sync is OFF
    3. Watch the MIDI monitor for 10–15 seconds
    4. Confirm: note onsets arrive at visibly irregular timing (some gaps short, some long — no two consecutive gaps look equal)
    5. Confirm: no stuck notes after the test

    **Test 2 — RND-09: RND SYNC ON + DAW stopped = internal clock grid**
    1. Turn RND SYNC ON; keep DAW stopped (or DAW Sync OFF)
    2. Watch MIDI monitor for 10–15 seconds
    3. Confirm: notes arrive at regular intervals matching the selected subdivision at the Free BPM setting
    4. Compare to the beat clock dot period on the Free BPM knob — they should match

    **Test 3 — RND-10: RND SYNC ON + DAW playing = DAW grid**
    1. Turn RND SYNC ON; start DAW playback with DAW Sync ON
    2. Watch MIDI monitor — notes should land on the beat grid at the selected subdivision
    3. Confirm: note timestamps align with DAW grid positions

    **Test 4 — Burst mechanics**
    1. Set Probability to 100%; Population = 2; Subdivision = 1/8 or 1/16
    2. Turn RND SYNC OFF, wait 5 seconds
    3. Confirm: you hear rapid clusters of notes (rolls/bursts) rather than single notes per trigger
    4. Set Probability to 0% → silence (no notes)
    5. Set Probability back to 50% → moderate density burst

    **Test 5 — MIDI safety at extreme settings**
    1. Set Population = 64, Probability = 100%, Subdivision = 1/32 or 1/64, RND SYNC OFF
    2. Let run for 10 seconds
    3. Turn the voice trigger source OFF or switch to Pad
    4. Confirm: no stuck notes remain; MIDI monitor shows all notes off

    Report: "approved" if all 5 tests pass, or describe any failures observed.
  </how-to-verify>
