## Task 5 — JoystickPad: Fix 5 — sticky offset model

**File: `Source/PluginEditor.h`** (or the JoystickPad class declaration within PluginEditor.cpp).

Add a member to JoystickPad:

```cpp
float looperOffsetX_ = 0.0f;  // sticky mouse offset when looper is playing with joy content
float looperOffsetY_ = 0.0f;
```

**`updateFromMouse()` changes** — currently writes directly to `proc_.joystickX/Y`.
Change to:

```cpp
void JoystickPad::updateFromMouse(const juce::MouseEvent& e)
{
    const float w  = static_cast<float>(getWidth());
    const float h  = static_cast<float>(getHeight());
    float nx = juce::jlimit(-1.0f, 1.0f, (e.x / w) * 2.0f - 1.0f);
    float ny = juce::jlimit(-1.0f, 1.0f, 1.0f - (e.y / h) * 2.0f);

    constexpr float kSnap = 0.05f;
    if (std::abs(nx) < kSnap) nx = 0.0f;
    if (std::abs(ny) < kSnap) ny = 0.0f;

    if (*proc_.apvts.getRawParameterValue("stickInvert") > 0.5f)
        std::swap(nx, ny);

    // When looper is playing with joystick content: set sticky offset, not direct joystick.
    // The audio thread adds this offset on top of looper playback (additive model).
    if (proc_.looperIsPlaying() && proc_.looperHasJoystickContent())
    {
        looperOffsetX_ = nx;  // nx IS the offset (center = 0 = no offset, by construction)
        looperOffsetY_ = ny;
        proc_.looperJoyOffsetX_.store(nx, std::memory_order_relaxed);
        proc_.looperJoyOffsetY_.store(ny, std::memory_order_relaxed);
    }
    else
    {
        proc_.joystickX.store(nx);
        proc_.joystickY.store(ny);
    }
    repaint();
}
```

**`mouseUp()` change** — currently just sets `mouseIsDown_ = false`. When looper is
playing with joystick content, do NOT reset the offset on mouseUp (sticky behavior):

```cpp
void JoystickPad::mouseUp(const juce::MouseEvent&)
{
    mouseIsDown_ = false;
    // Sticky offset model: do NOT clear looperJoyOffsetX_/Y_ on mouseUp.
    // The offset persists until the user clicks center or double-clicks.
}
```

**`mouseDoubleClick()` change** — reset offset to zero on double-click:

```cpp
void JoystickPad::mouseDoubleClick(const juce::MouseEvent& /*e*/)
{
    proc_.joystickX.store(0.0f);
    proc_.joystickY.store(0.0f);
    looperOffsetX_ = 0.0f;
    looperOffsetY_ = 0.0f;
    proc_.looperJoyOffsetX_.store(0.0f, std::memory_order_relaxed);
    proc_.looperJoyOffsetY_.store(0.0f, std::memory_order_relaxed);
    repaint();
}
```

**Spring-back behavior change** — in `timerCallback()`, the spring-damper block runs when
`!mouseIsDown_` (line ~1131). This spring back to the looper's cursor position is now
correct because `cx`/`cy` are computed from `joystickX/Y` which includes the offset.
No change needed to spring logic — it already targets the effective position.

However, the spring should NOT apply when looper is playing with joy content AND the
offset is sticky (since we don't want the cursor to spring back when mouse is released).
Suppress the spring and snap display cursor to effective position:

In the spring-damper block, before the existing `if (mouseIsDown_)` check:

```cpp
// When looper is playing with joystick content and mouse is up:
// cursor target = looper position + sticky offset (already in joystickX/Y via processBlock).
// No spring needed — just snap display cursor to target to avoid ghost oscillation.
const bool looperJoyMode = proc_.looperIsPlaying() && proc_.looperHasJoystickContent();
if (looperJoyMode && !mouseIsDown_)
{
    displayCx_  = cx;  // cx already reflects looper + offset
    displayCy_  = cy;
    springVelX_ = 0.0f;
    springVelY_ = 0.0f;
}
else if (mouseIsDown_)
{
    displayCx_  = cx;
    displayCy_  = cy;
    springVelX_ = 0.0f;
    springVelY_ = 0.0f;
}
else
{
    // existing spring-damper code (unchanged)
    constexpr float dt = 1.0f / 60.0f;
    constexpr float k  = 320.0f;
    constexpr float d  = 16.0f;
    const float ax = -k * (displayCx_ - cx) - d * springVelX_;
    const float ay = -k * (displayCy_ - cy) - d * springVelY_;
    springVelX_ += ax * dt;
    springVelY_ += ay * dt;
    displayCx_  += springVelX_ * dt;
    displayCy_  += springVelY_ * dt;
}
```
