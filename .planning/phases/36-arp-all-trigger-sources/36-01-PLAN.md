# Phase 36 Plan 01: Arp + All Trigger Sources

## Goal

Remove the 5-line block in `processBlock` that forces all voice trigger sources to
`TriggerSource::TouchPlate` when the arp is active. After this fix:

- **Joystick** voices fire direct notes normally; arp skips them while the joystick note is
  sounding and fires them in the gaps (gate-closed windows).
- **Random Free/Hold** voices fire direct notes normally; arp skips them during the random
  gate-length window and fires them between random events.
- **TouchPlate** voices behave identically to before (no regression).
- No double notes, no hanging notes — the existing `isGateOpen → skip` check at line 1884
  and the arp kill-on-disable block already handle all cases correctly.

## Task 2 — checkpoint:human-verify

Build and install, then verify all four success criteria in the DAW:

**SC1 — Joystick + Arp**
1. Set all 4 voices to trigger source: Joystick
2. Enable the arp (any rate, e.g. 1/8)
3. Move the joystick outside the dead zone
4. PASS: Joystick notes fire directly AND arp steps fire for voices when joystick gate
   is not active (between joystick settle cycles). No forced fallback to pad mode.

**SC2 — Random Free + Arp**
1. Set all 4 voices to trigger source: Random Free
2. Enable arp, set a slow rate (e.g. 1/4)
3. Let random events fire (DAW playing or free BPM running)
4. PASS: Arp steps fire at its rate; random direct notes also fire when random gate opens.
   No forced fallback to pad mode. No double notes.

**SC3 — TouchPlate + Arp (regression)**
1. Set all 4 voices to trigger source: TouchPlate (default)
2. Enable arp
3. Press and hold pad 1; observe arp stepping through voices 2/3/4
4. PASS: Held pad voice is skipped by arp (pad note sounds directly). Behavior identical
   to before this fix.

**SC4 — No hanging notes on arp disable**
1. With arp running (any source), an arp note is sounding
2. Turn arp off mid-phrase
3. PASS: The active arp note receives a note-off immediately. No stuck notes.
