Three changes to PluginEditor.cpp.

**Change A: timerCallback — beat pulse read and decay**

In timerCallback(), in the BPM display update block (near the existing `bpmDisplayLabel_.setText(...)` call), add beat pulse handling immediately after or before the BPM text update:

```cpp
// Beat clock pulse — read-and-reset beatOccurred_ then decay
if (proc_.beatOccurred_.exchange(false, std::memory_order_relaxed))
    beatPulse_ = 1.0f;
beatPulse_ = juce::jmax(0.0f, beatPulse_ - 0.11f);  // ~9 frames to zero at 30 Hz

// Trigger repaint of the BPM label area to update the dot
if (beatPulse_ > 0.0f || proc_.beatOccurred_.load(std::memory_order_relaxed))
    repaint(bpmDisplayLabel_.getBounds().expanded(16, 6));
```

Note: The repaint above uses a slightly expanded region to include the dot (8px diameter, positioned 4px right of label). Use `expanded(16, 6)` to cover the dot area without full-editor repaint.

---

**Change B: paint() — beat dot drawing**

In PluginEditor::paint(), at the END of the paint function (after all existing drawing including the LFO panel drawing added in Plan 02), add:

```cpp
// ── Beat clock dot ─────────────────────────────────────────────────────────
// Drawn 4px to the right of the BPM display label, vertically centred
if (!bpmDisplayLabel_.getBounds().isEmpty())
{
    const auto labelB = bpmDisplayLabel_.getBounds();
    const float dotX = (float)(labelB.getRight() + 4);
    const float dotY = (float)(labelB.getCentreY()) - 4.0f;

    // Interpolate: at beatPulse_=1.0 → bright cyan; at 0.0 → dim grey
    const juce::Colour dotClr = Clr::accent
        .interpolatedWith(Clr::highlight.brighter(0.5f), beatPulse_)
        .withAlpha(0.25f + 0.75f * beatPulse_);
    g.setColour(dotClr);
    g.fillEllipse(dotX, dotY, 8.0f, 8.0f);
}
```

Note: `Clr::accent` is `0xFF1E3A6E` (mid-blue) — dim state. `Clr::highlight.brighter(0.5f)` is brighter pink-red. Per CONTEXT.md the on-beat colour should be `Clr::accent` cyan — but looking at the project's colour table, `Clr::accent` is mid-blue, not cyan. The CONTEXT.md says "fills with Clr::accent (cyan)". Use the CONTEXT.md intent: bright accent. Use `juce::Colour(0xFF00E5FF)` as the bright beat colour (bright cyan), interpolating from `Clr::gateOff` (dim state):

```cpp
const juce::Colour offClr = Clr::gateOff;
const juce::Colour onClr  = juce::Colour(0xFF00E5FF);  // bright cyan
const juce::Colour dotClr = offClr.interpolatedWith(onClr, beatPulse_)
                                   .withAlpha(0.3f + 0.7f * beatPulse_);
```

---

**Change C: JoystickPad::paint() — LFO-modulated position**

In JoystickPad::paint(), replace lines 353-354 (the cursor dot position calculation) which currently read:
```cpp
const float cx = (proc_.joystickX.load() + 1.0f) * 0.5f * b.getWidth()  + b.getX();
const float cy = (1.0f - (proc_.joystickY.load() + 1.0f) * 0.5f) * b.getHeight() + b.getY();
```

With:
```cpp
// Use LFO-modulated position when either LFO is active
const bool lfoXActive = *proc_.apvts.getRawParameterValue("lfoXEnabled") > 0.5f
                     && *proc_.apvts.getRawParameterValue("lfoXLevel")   > 0.0f;
const bool lfoYActive = *proc_.apvts.getRawParameterValue("lfoYEnabled") > 0.5f
                     && *proc_.apvts.getRawParameterValue("lfoYLevel")   > 0.0f;
const float displayX = (lfoXActive || lfoYActive)
    ? proc_.modulatedJoyX_.load(std::memory_order_relaxed)
    : proc_.joystickX.load(std::memory_order_relaxed);
const float displayY = (lfoXActive || lfoYActive)
    ? proc_.modulatedJoyY_.load(std::memory_order_relaxed)
    : proc_.joystickY.load(std::memory_order_relaxed);

const float cx = (displayX + 1.0f) * 0.5f * b.getWidth()  + b.getX();
const float cy = (1.0f - (displayY + 1.0f) * 0.5f) * b.getHeight() + b.getY();
```

Note: `getRawParameterValue()` returns `std::atomic<float>*`. Calling `*ptr` on the message thread (JoystickPad::paint runs on message thread) is safe per JUCE design — these are APVTS raw values, not audio-thread-only data.
