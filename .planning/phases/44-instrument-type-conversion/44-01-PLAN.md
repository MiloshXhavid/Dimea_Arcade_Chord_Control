### Task 3 — PluginProcessor.cpp: enable output bus in constructor

Edit `Source/PluginProcessor.cpp` constructor BusesProperties (lines 369–371):

```cpp

      .withInput ("Input",  juce::AudioChannelSet::stereo(), false)
```

- Change `.withOutput(...)` third arg from `false` to `true` (enable the output bus)

### Task 4 — PluginProcessor.cpp: update isBusesLayoutSupported

Edit `Source/PluginProcessor.cpp` function `isBusesLayoutSupported` (lines 514–521):

```cpp
// OLD:
bool PluginProcessor::isBusesLayoutSupported(const BusesLayout& layouts) const
{
    // MIDI effect: buses must be disabled (0 ch) or stereo passthrough (2 ch).
    // Ableton deactivates audio buses before loading on a MIDI track;
    // accepting both layouts prevents kResultFalse from activateBus().
    const int numIn  = layouts.getMainInputChannels();
    const int numOut = layouts.getMainOutputChannels();
    return (numIn == 0 && numOut == 0) || (numIn == 2 && numOut == 2);
}

// NEW:
bool PluginProcessor::isBusesLayoutSupported(const BusesLayout& layouts) const
{
    // Instrument: no audio input, stereo output.
    // Accept numOut == 0 for DAWs that don't need audio from a MIDI generator.
    const int numIn  = layouts.getMainInputChannels();
    const int numOut = layouts.getMainOutputChannels();
    return numIn == 0 && (numOut == 0 || numOut == 2);
}
```

## Must-Haves (goal-backward)

For Phase 44's goal ("universal DAW compatibility via instrument type"):
- `IS_MIDI_EFFECT FALSE` in CMakeLists — if true, JUCE VST3 wrapper disables audio buses regardless of C++ code
- `isMidiEffect()` returns `false` — if true, Ableton still treats it as MIDI FX
- Output bus enabled (`true`) in constructor — if `false`, host won't see audio output bus → won't offer instrument slot in some DAWs
- `isBusesLayoutSupported` accepts numOut=2 — if it rejects stereo, host deactivates the plugin on instrument track
