### Task 2 — Checkpoint: Human DAW verification

**This is a human-verify checkpoint. Stop and ask the user to perform the following tests.**

---

#### Test 1 — Ableton (regression: instrument slot + MIDI output routing)

1. Open Ableton
2. Create a new MIDI track
3. In the instrument slot, browse for "Arcade Chord Control (BETA-Test)" — it should appear under **Instruments**, NOT under Audio Effects
4. Insert the plugin in the instrument slot
5. Create a second MIDI track, set its **MIDI From** to the first track (ChordJoystick output)
6. Insert any synth on the second track
7. Play joystick + press pad → confirm chords sound on the synth
8. Confirm audio output of the ChordJoystick track is silent

**Pass criteria:**
- Plugin found in Instruments list (not Audio Effects)
- MIDI routing to second track works
- Chords play correctly
- No audio output from ChordJoystick track

---

#### Test 2 — Reaper (regression: stacked on same track)

1. Open Reaper
2. Create a MIDI track
3. Add "Arcade Chord Control (BETA-Test)" as first FX on the track
4. Add any synth as second FX on the same track
5. Play joystick → confirm chords sound
6. Confirm no regression from previous behavior

**Pass criteria:**
- Plugin loads without error
- MIDI flows from ChordJoystick to synth on same track
- All features (LFO, arp, looper, gamepad) work as before

---

#### Test 3 — Cakewalk (primary motivation)

1. Open Cakewalk
2. Create a new project with a Synth track
3. In the synth/instrument browser, search for "Arcade Chord Control" — it should now appear as an instrument
4. Insert it in an instrument track
5. Route its MIDI output to a second instrument (or use it standalone)
6. Confirm plugin UI opens correctly

**Pass criteria:**
- Plugin appears in Cakewalk's instrument list (not invisible as before)
- Plugin UI opens
- No crash on load or on close (C0000005 fix from v1.8 already applied)

---

#### Test 4 — Feature regression (any DAW)

With plugin loaded as instrument:
- [ ] All 4 voice pads trigger MIDI notes on correct channels
- [ ] Joystick changes pitch correctly
- [ ] LFO modulates as expected
- [ ] Arp fires steps correctly
- [ ] Looper records and plays back
- [ ] Gamepad (if connected) triggers voices and looper
- [ ] Preset save and load works — parameters restore correctly
- [ ] Existing preset files (.vstpreset or DAW project state) load without errors

## Failure Handling

**If Cakewalk test fails (plugin still not detected):**
- Verify VST3 category in the built DLL: open the .vst3 bundle's `moduleinfo.json` and confirm `"subcategories": ["Instrument", "Fx"]`
- If category wrong: CMake reconfigure was not run — delete `build/CMakeCache.txt`, re-run `build32.ps1`

**If Ableton test fails (plugin in wrong slot):**
- Confirm `isMidiEffect()` returns `false` in built DLL
- Check if Ableton cached the old plugin state — restart Ableton, rescan VST3

**If any feature regression is found:**
- The code changes are purely structural (bus config + flags) — any regression points to an unexpected JUCE behavior
- Revert plan 44-01 changes and investigate `isBusesLayoutSupported` return value for the specific host
