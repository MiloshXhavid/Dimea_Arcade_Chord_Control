# Summary: Modulation Polish and CC Routing

**one_liner**: Fixed MOD FIX liveGamepad dispatch (8 missing LFO target cases) and expanded filter CC routing from 4 to 18 named CCs.

## What Was Done

### 1. MOD FIX liveGamepad offset fix (commit `2f2f5e3`)
- **Problem**: MOD FIX knob had no effect on LFO Phase, Level, or cross-axis Freq/Phase/Level while a gamepad was connected (even with stick centered). Only direct Freq (case 4) worked correctly.
- **Root cause**: `if (liveGamepad)` block was missing `(offset/50)*sensitivity` in 8 of 10 LFO target cases. The no-gamepad branch at line 2413 had all cases correct.
- **Fix**: Added `(xOffset/50)*scale` and `(yOffset/50)*scale` to xMode cases 5, 6, 8, 9, 10 and yMode cases 5, 6, 8, 9, 10 in `PluginProcessor.cpp`.

### 2. Expand CC routing to 18 named CCs (commit `ddaec4f`)
- **Problem**: filterXMode/filterYMode offered only 4 hardcoded CCs (CC74, CC12, CC71, CC76); users needed access to the full CC1–CC93 set already available on lfoXCcDest.
- **Fix**: Replaced all hardcoded ternary chains with `kFilterCcNums[mode - kNumLfoTargetModes]` lookup (18-entry array). Updated `PluginEditor.cpp` combo population. LFO/Gate modes shift to indices 18–24.
- **Result**: filterXMode default=16 (CC71), filterYMode default=19 (CC74) — same effective routing as before, now user-configurable across 18 CCs.

- `Source/PluginEditor.cpp` — CC combo population + mode index adjustments
