### Human Verification Required

#### 1. Mini shrink via header button

**Test:** In Full view, click the ◱ button (far-right of header strip, ~20px icon). Move mouse first to confirm hover brightness changes.
**Expected:** Window shrinks to a square matching the pad size at current scale factor (~308px at 1x, ~231px at 0.75x). Only the joystick pad is visible — no header bar, no controls, no footer.
**Why human:** Window resize and child visibility toggling are runtime behaviors.

#### 2. Maxi expand via header button

**Test:** From Full view, click the ⛶ button (next to ◱ in header). Verify the window fills the display.
**Expected:** Window expands to a square equal to the smaller dimension of the usable display area. JoystickPad fills the entire window.
**Why human:** `juce::Desktop::getDisplays()` requires a running DAW environment with a physical display.

#### 3. Overlay ↩ return button

**Test:** Enter Mini or Maxi mode, then click the small semi-transparent button in the top-right corner of the pad.
**Expected:** Window returns to Full view with all controls visible. Gamepad OPTION indicator resets to dim "OPTION" text (mode 0).
**Why human:** JoystickPad hitTest passthrough and mouse event routing to parent require live test.

#### 4. Starfield and cursor animation in compact modes

**Test:** Enter Mini or Maxi mode. Move the mouse/joystick inside the pad.
**Expected:** Spring-damper cursor moves smoothly; starfield animates; warp stars, crosshair lines, and burst particles all render. Stars scale proportionally in Maxi (not tiny).
**Why human:** Particle and animation rendering requires visual inspection.

#### 5. Scale preservation across Mini visit

**Test:** Resize plugin to 0.75x in Full view. Enter Mini mode. Return to Full via overlay button. Close and reopen.
**Expected:** Full view reopens at 0.75x (840×630) — not at the mini window dimensions.
**Why human:** `savedUiScale_` guard correctness requires a resize-then-compact-then-restore round-trip.

#### 6. Opens Full after session ends in Mini

**Test:** Enter Mini mode, then close the plugin instance in the DAW. Reload the project.
**Expected:** Plugin opens in Full view — `WindowMode::Full` is the constructor default and is never serialised.
**Why human:** DAW load/unload cycle required to test session persistence behavior.

#### 7. Gamepad modes 3/4 and D-pad BPM control

**Test:** With a controller connected, cycle the Option button to mode 3 (OPTION indicator shows "MINI" in teal). Press D-pad Up/Down.
**Expected:** Window enters Mini on mode 3, Maxi on mode 4. D-pad Up/Down changes BPM (not octave steps) in modes 3 and 4.
**Why human:** Requires physical gamepad or emulation; D-pad routing change is conditional logic.
